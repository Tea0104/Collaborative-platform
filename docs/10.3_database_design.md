# 10.3 数据库设计文档

## 10.3.1 数据库概述
- 数据库类型：SQLite
- 访问方式：Python `sqlite3` + 原生 SQL
- 初始化方式：应用启动时执行 `init_database()` 建表；数据库为空时执行 `seed_demo_data_if_empty()` 写入示例数据。
- 代码位置：`server/app_factory.py:create_app`、`server/db.py:init_database`、`server/db.py:seed_demo_data_if_empty`

## 10.3.2 表结构设计

### 表：`user`
- 用途：存储学生和企业用户的账号与基础信息。
- 字段：
  - `user_id`：用户ID。
  - `username`：登录用户名。
  - `password_hash`：密码哈希。
  - `user_type`：用户类型，区分学生/企业。
  - `real_name`：真实姓名或企业名称。
  - `school_company`：学校或公司信息。
  - `skill_tags`：技能标签。
  - `contact`：联系方式。
  - `status`：账号状态。
  - `create_time`：注册时间。
  - `last_login`：最近登录时间。

### 表：`project`
- 用途：存储企业发布的项目。
- 字段：
  - `project_id`：项目ID。
  - `project_name`：项目名称。
  - `description`：项目描述。
  - `publisher_id`：项目发布者，对应企业用户。
  - `project_status`：项目状态。
  - `publish_time`：发布时间。
  - `deadline`：项目截止时间。
  - `result_url`：成果链接。
  - `expected_market`：预期市场。
  - `work_mode`：协作方式。
  - `participant_count`：参与人数描述。
  - `company`：公司名称。

### 表：`role`
- 用途：存储项目下拆解出的岗位。
- 字段：
  - `role_id`：岗位ID。
  - `project_id`：所属项目ID。
  - `role_name`：岗位名称。
  - `task_desc`：岗位任务说明。
  - `skill_require`：技能要求。
  - `limit_num`：招募人数上限。
  - `join_num`：已加入人数。
  - `role_status`：岗位状态。
  - `task_deadline`：岗位截止时间。

### 表：`role_application`
- 用途：存储学生的岗位申请记录。
- 字段：
  - `application_id`：申请ID。
  - `role_id`：申请的岗位ID。
  - `project_id`：所属项目ID。
  - `student_id`：申请学生ID。
  - `motivation`：申请动机。
  - `status`：申请状态。
  - `apply_time`：申请时间。
  - `update_time`：审核或更新的时间。

### 表：`role_feedback`
- 用途：存储岗位反馈或成果提交记录。
- 字段：
  - `feedback_id`：反馈ID。
  - `project_id`：所属项目ID。
  - `role_id`：所属岗位ID。
  - `user_id`：提交用户ID。
  - `content`：反馈内容。
  - `evidence_url`：附件或证明链接。
  - `status`：反馈状态。
  - `created_at`：提交时间。

### 表：`auth_tokens`
- 用途：存储登录后的 Bearer Token。
- 字段：
  - `token`：登录令牌。
  - `user_id`：令牌所属用户ID。
  - `created_at`：令牌创建时间。

## 10.3.3 表关系说明
- `user` 与 `project`：一个企业用户可以发布多个项目。
- `project` 与 `role`：一个项目可以包含多个岗位。
- `user` 与 `role_application`：一个学生可以提交多个岗位申请。
- `role` 与 `role_application`：一个岗位可以收到多个申请。
- `project` 与 `role_application`：一个项目下可以有多条申请记录。
- `project` 与 `role_feedback`：一个项目可以有多条反馈记录。
- `role` 与 `role_feedback`：一个岗位可以有多条反馈记录。
- `user` 与 `role_feedback`：一个用户可以提交多条反馈。
- `user` 与 `auth_tokens`：一个用户可以对应多个 Token。
